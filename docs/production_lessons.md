# FEP 영상 제작 실패 경험 & 절차 가이드

> **목적**: EP.01 제작 과정에서 축적된 실패/성공 경험을 체계적으로 기록하여,
> EP.02~ 후속 에피소드에서 같은 실수를 반복하지 않고 효율적으로 제작하기 위한 참조 문서.
> 
> **⚠️ 새 에피소드 시작 시 반드시 이 문서를 먼저 읽을 것.**

---

## 1. 일러스트 생성

### 1-1. txt2img 단독 캐릭터 생성 → 실패

**문제**: txt2img로 캐릭터를 생성하면 매번 얼굴, 체형, 의상이 달라짐. 같은 프롬프트로도 일관성 유지 불가.

**시도**: 5장 생성 후 비교 → 캐릭터 시트와 불일치 판정 → 전량 폐기 (`characters_txt2img_backup/` 격리)

**해결**: **캐릭터 시트 img2img 참조 필수** (denoise 0.3~0.5). Style B는 txt2img 사용하되 캐릭터 시트를 프롬프트에 상세 기술.

**교훈**: 캐릭터 시트 없이 씬 생성 절대 금지. 에피소드 시작 시 캐릭터 시트 먼저 확보.

---

### 1-2. 완성 씬 일괄 생성 → 실패

**문제**: 배경+캐릭터를 한 이미지로 생성하면:
- 캐릭터 위치/크기 제어 불가
- 같은 장소 다른 씬에서 배경 불일치
- LP 애니메이션 적용 시 배경까지 변형
- 캐릭터 재사용 불가

**해결**: **레이어 분리 생성** — 배경/캐릭터/소품을 독립 생성 후 Pillow 합성.

**교훈**: EP.02부터 "완성 씬 일괄 생성"은 완전 폐기. 배경 마스터 + 캐릭터 포즈 라이브러리 → 합성.

---

### 1-3. 어두운 톤 프롬프트 → 검은 화면

**문제**: `"dark atmosphere"`, `"shadow"`, `"black background"` 등 프롬프트 → AI가 거의 검은 화면 생성 (scene_13 사례).

**해결**: "어둡다"를 직접 쓰지 않고 구체적 색상으로 대체.
- ❌ `"dark moody atmosphere"`
- ✅ `"muted purple-gray tones, dim warm lamplight from right"`

---

### 1-4. 로케이션 색감 앰버/석양 → 부적합

**문제**: FLUX 기본 생성이 앰버/골든아워 톤으로 치우침. 5개 로케이션 전부 석양 느낌.

**해결**: 네거티브에 `"orange tint, amber, golden hour, sunset, sepia"` 추가. BG_STYLE을 `"bright clean neutral white lighting"`으로 고정.

**교훈**: 로케이션 생성 시 네거티브 프롬프트에 앰버 계열 반드시 배제.

---

### 1-5. FP8 모델 MPS 비호환

**문제**: `flux1-kontext-dev-fp8` 모델을 Apple Silicon MPS에서 로드 시 크래시.

**해결**: **GGUF Q8** 사용 (`flux1-kontext-dev-Q8_0.gguf` + `UnetLoaderGGUF` 노드).

---

### 1-6. ComfyUI CLIP 모델명 오류

**문제**: API 호출 시 `t5xxl_fp16.safetensors` → 400 에러 (파일 존재하지 않음).

**해결**: 정확한 모델명 `t5xxl_fp8_e4m3fn_scaled.safetensors` 사용.

**교훈**: ComfyUI API 스크립트 작성 시 `models/clip/` 디렉토리의 실제 파일명 확인 필수.

---

## 2. 알파 추출

### 2-1. 글로벌 threshold 흰 배경 제거 → 흰 옷 투명화

**문제**: 전체 이미지에서 "밝기 > threshold" 픽셀을 투명화 → 흰 옷, 흰 소품까지 제거.

**해결**: **Flood Fill 방식** — 이미지 가장자리에서 연결된 흰색만 배경 판정. 내부의 흰색(옷, 눈 등)은 보존.

---

### 2-2. rembg 복잡 배경 분리 → 품질 열화

**문제**: 복잡한 씬에서 rembg로 캐릭터 분리 시 머리카락 경계 거칠게 잘림, 거울 반사/말풍선도 전경으로 분류.

**해결**: 
- 솔리드 배경(#00FF00)에서 캐릭터 생성 → 크로마키 추출 (품질 우수)
- rembg는 솔리드 배경 전용으로만 사용 (복잡 배경 금지)

---

### 2-3. 크로마키 green spill (초록끼 잔류)

**문제**: 그린스크린 크로마키 후 머리카락 갈라진 부분에 초록색 잔류.

**해결**: `despill_green()` 함수 — 초록 과잉 픽셀을 감지하고 주변 클린 픽셀 색상으로 블렌딩.

---

### 2-4. Style B 알파 크기 문제 (1344×768 전체 캔버스)

**문제**: Style B 캐릭터 알파가 씬 전체 크기(1344×768)로 생성되어, 합성 시 투명 영역이 포함된 거대 이미지로 처리 → 오버플로우/위치 오류.

**해결**: **auto_crop** — numpy로 투명 영역 자동 제거, 모든 캐릭터에 기본 적용 (default=True).

---

## 3. FFmpeg / 영상 빌드

### 3-1. `-shortest` 싱크 오버런

**문제**: `-loop 1` + zoompan + `-shortest` 조합에서 영상이 오디오보다 1~2초 길게 생성.

**해결**: `-shortest` 대신 반드시 `-t {duration}` 명시.

---

### 3-2. concat glob에 임시 파일 포함 → 영상 부풀림

**문제**: `_seg`, `_cut`, `_bcut` 등 임시 파일이 glob 패턴에 포함되어 이중 concat → 133초 부풀림 사고.

**해결**: glob 패턴에서 임시 파일 접두사 명시적 제외. `if "_temp_" not in str(f)` 가드.

---

### 3-3. 비트 클립 오디오 파라미터 불일치

**문제**: 메인 클립(24000Hz/mono) + 비트 클립(44100Hz/stereo) → concat 시 20분 오류 영상.

**해결**: 비트 클립도 동일 파라미터 `-ar 24000 -ac 1`로 생성.

**교훈**: concat 대상 클립의 오디오 샘플레이트/채널 수는 반드시 일치.

---

### 3-4. FFmpeg drawtext 한글 미지원

**문제**: macOS FFmpeg `drawtext` 필터에서 한글 폰트 렌더링 실패/깨짐.

**해결**: Pillow 기반 텍스트 오버레이로 전면 교체. PNG 시퀀스 생성 → FFmpeg overlay.

---

### 3-5. zoompan + 비디오 입력 비호환

**문제**: FFmpeg zoompan 필터는 이미지 입력만 지원. LP 영상(비디오)에 적용 불가.

**해결**: `scale+crop` 조합으로 미세 줌 대체.

---

## 4. LivePortrait

### 4-1. LP CLI boolean 플래그 파싱

**문제**: `--flag_pasteback true` → "Unrecognized arguments: true" 에러.

**해결**: `--flag_pasteback` 단독 사용 (값 없이). argparse boolean flag.

---

### 4-2. LP named pkl 템플릿 데이터 누락

**문제**: `wink.pkl`, `shy.pkl` 등 named 템플릿에 `eye`/`lip` 키 없음 → KeyError 크래시.

**해결**: `pipeline.py` 패치 — 누락 키에 대해 zeros 폴백.

---

### 4-3. LP 단순 만화체 얼굴 미감지

**문제**: Style A (2.5등신 SD체) 캐릭터의 얼굴을 LP가 감지하지 못함.

**해결**: Style B (6-7등신 리얼체)만 LP 적용. Style A는 브리딩 폴백. 뚜렷한 얼굴 특징이 있는 그림체에서만 LP 작동.

**EP.02 검토**: Kling AI / Runway Gen-3 등 클라우드 솔루션으로 만화체 애니메이션 가능 여부 재평가.

---

### 4-4. LP + ComfyUI GPU 경합

**문제**: LP와 ComfyUI 모두 MPS GPU 사용. 동시 실행 시 메모리 폭주 / 크래시.

**해결**: **순차 실행** — LP 배치 완료 후 ComfyUI 실행. GPU 작업은 절대 동시 실행 금지.

---

### 4-5. LP 영상 배경: 그린스크린 출력

**특성**: LP 입력이 투명 배경(알파) PNG라도, 출력은 MP4(알파 미지원) → 투명 영역이 그린으로 채워짐.

**활용**: FFmpeg `chromakey=0x30C010:0.12:0.02`로 그린 배경 제거 → BG 위에 overlay 합성.

**⚠️ 파라미터 교훈**: 초기값 `0x40E810:0.25:0.08`은 similarity/blend가 너무 높아 캐릭터가 반투명+초록끼 발생. 낮은 값(`0.12:0.02`)이 불투명하고 깨끗. 4종 비교 테스트 후 확정.

---

## 5. 말풍선 / 텍스트 오버레이

### 5-1. Pillow 기하학적 말풍선 → 품질 부적합

**문제**: Pillow로 arc/polygon 조합으로 말풍선 생성 → 각지고 거칠고 어색한 모양. 사용자 피드백: "지금처럼 이렇게 할거면 안넣는게 낫겠어"

**해결**: **ComfyUI FLUX로 말풍선 에셋 생성** → Flood Fill 알파 추출 → 재사용 가능 PNG.

**교훈**: UI/비주얼 에셋은 AI 생성이 Pillow 코드보다 훨씬 자연스러움.

---

### 5-2. 말풍선 위치가 캐릭터와 무관

**문제**: "left_bubble", "right_bubble" 등 일반적 위치 → 실제 캐릭터 위치와 불일치.

**해결 방향**: 포트레이트 모드 레시피의 `position` 값을 참조하여 캐릭터 상단에 배치. (build_video_v3.py에 반영 예정)

---

## 6. 메모리 / 시스템

### 6-1. ComfyUI 모델 로드 시 메모리 급락

**현상**: FLUX GGUF 모델 첫 로드 시 메모리 free 90% → 37~42%로 하락. 이후 안정.

**대처**: 일시적 현상이므로 패닉 불필요. 다만 LP와 동시 실행은 위험.

---

### 6-2. GPU 워크로드 후 메모리 미반환

**문제**: ComfyUI/LP 종료 후에도 MPS 메모리가 즉시 반환되지 않음.

**해결**: ComfyUI는 작업 후 즉시 종료 권장 (`kill`). 장시간 idle 상태로 두지 않기.

---
